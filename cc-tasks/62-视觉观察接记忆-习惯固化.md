# Brief 62 · 视觉观察 → 习惯固化（阶段 1.5，带准入闸门）

> ⛔ **准入条件（全部满足才允许执行本单，否则退回）**：
> 1. Brief 56 shadow 模式已运行 ≥ 14 天；
> 2. `scripts/audit_visual_trace.py` 人工抽查 ≥ 50 条 caption，幻觉率 < 10%
>    （描述与实际不符即算幻觉）；
> 3. `dropped=sensitive` 漏判 0 例（抽查中无敏感画面被正常记录）；
> 4. scene/activity 分布与用户真实作息大体一致（用户本人确认）。
> 条件不满足时的调整路径见 `cc-tasks/00b-交接20260712-四系列现状与后续指南.md` §3。
>
> 依赖：56（visual_trace）、45（习惯写入走 ADD/UPDATE/NOOP 冲突裁决入口）。

## 1. 定位

把 shadow 观察流聚合成"记住的习惯"，写入 profile 的 `habit` tag facts——
**不写 identity**（identity 是行为模式层，由 fixation 链管），**不写 episodic**
（单帧观察不是事件）。habit facts 已有现成的层 5_profile_pref recency 门控消费端。

## 2. 聚合触发器（core/scheduler/triggers/habit_consolidate.py）

每日一次（`stamp_trigger()`，不发言）：

1. **规则聚合（零 LLM）**：读近 14 天 visual_trace，按 `(scene, activity) × 时段桶`
   （0-6/6-12/12-18/18-24）做直方图；仅取 `confidence ≥ 0.6` 的条目。
2. **稳定模式候选**：同一 `(scene, activity, 时段)` 组合出现 ≥ 8 天 → 候选。
3. **固化（一次 LLM 调用）**：候选 + 现有 habit facts → 生成一条 ≤30 字习惯描述
   （"她通常在深夜看视频"这类，第三人称，`user_name` 插值），经 **Brief 45 的
   extract_and_update 冲突裁决入口**写入（op=add/update/noop——作息变了自动 update，
   已存在自动 noop）。
4. 单日新增/更新 ≤ 1 条（常量）；WriteEnvelope 为 stamp_trigger；provenance 由 45 入口自带。

## 3. 红线

- caption 原文永不入任何记忆层——只有聚合后的习惯描述入 profile。
- 敏感兜底：习惯描述 LLM prompt 明确禁止涉及具体内容（"在看什么视频"不写，
  只写"在看视频"）；输出含具体作品名/人名 → 丢弃本条（正则 + 长度双闸）。
- 习惯可经 G2 显式遗忘删除（profile facts 删除端点现成）。

## 4. 测试

1. 准入闸门自检：trace 不足 14 天 → 触发器 no-op 并 WARN（代码级兜底，不只靠人）。
2. 直方图聚合：8 天阈值、confidence 过滤、时段桶边界。
3. 固化经 45 入口：重复习惯 noop、作息变更 update（种子对照）。
4. 单日上限 1 条；敏感描述丢弃分支。
5. `is_test` envelope 拒写。
6. `pytest -n auto` 新增测试。

## 5. 不做什么

- 不做实时"你现在在干嘛"注入（screen_awareness 3.9 层已有，职责不同）。
- 不写 identity / episodic / hidden_state。
- 不做习惯的主动评论 proposer（他"知道"了自然会在对话里带出，先观察）。
