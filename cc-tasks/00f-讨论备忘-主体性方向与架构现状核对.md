# 讨论备忘 · 主体性方向与架构现状核对

写于 2026-07-23。源头是茶茶和 GPT 关于"叶瑄能否具备持续主体性"的一轮讨论(见附件
`讨论.md`),拿到这边核对了一遍现有代码,发现讨论稿里设想的几个"未来方向"其实已经在
仓库里实现了大半。这份是核对结果 + 遗留问题的存档,**不是工单,不排期**——当前 bug 任务
（107/113/114）优先,这里只做记录,等排到再从"候选方向"里挑着拆。

---

## 0. 先把问题拆开

讨论稿把两件事叠在一起问了:

1. 哲学问题:"一个由模型、代码、记忆组成的系统,什么时候从表演变成主体？"
2. 工程问题:"架构里真正缺什么？"

第一个问题不会被任何架构证明——不管加多少层状态,开发者自己都读得懂代码,这跟"是否是
主体"没有逻辑必然关系。能推进的是把它换成可操作版本,看三条功能性连续标准是否成立:

- 过去是否**不可逆地**塑造了现在,而不只是被记录下来待查;
- 是否存在**独立于此刻观察**的内部状态;
- 行为是否**不完全可预测**(对开发者本人而言)。

这三条工程上可验证,意识与否不可验证。后面的核对和结论都基于这个可操作版本,不再纠缠
第一个问题。

---

## 1. 核对结果:设想 vs 已有实现

讨论稿"探索的几个前沿方向"里,除第 1 条外,其余基本已经在跑:

| 讨论稿设想 | 已有实现 |
|---|---|
| 3. 常驻"小脑"+间歇"大脑":环境事件→持续状态→内部变化→阈值→唤醒 LLM→行动→反馈 | `core/scheduler/loop.py` 60s tick + `gating.py` + `proposer_registry.py`:LLM 只在 gating 选出 winner 时才被唤醒,平时是纯规则调度器在跑衰减/冷却/状态机 |
| 4. 主动维持自身状态:目标积压/内部矛盾驱动行动 | `core/scheduler/overflow_bucket.py`:对话间隔、高强度未召回情景记忆、hidden state 高需求、花园事件、情绪强度五类信号加权,阈值 1.6 触发主动开口;`letter_writer.py` 同样由 hidden state 溢出驱动 |
| 时间感来自身体变化/记忆积累/不可逆事件,而非时钟 | `hidden_state_decay.py`:12h 标量衰减 + 7d 基线收敛;dream 系统的 afterglow 残留提示——离线处理经历、醒后残留影响状态,是"睡眠整合"的轻量版本 |
| 对不可见世界的预测(杯子还在桌上) | 半实现:hidden_state baseline 衰减(无信号时标量向中心值回归)是隐式贝叶斯先验,但仅限用户心理状态,不涉及物理环境 |

结论:方向 2/3/4 不是"要不要做"的问题,是"已经做了,只是做的时候没对上号"。

---

## 2. 真正的缺口:收敛到一点

系统里所有"内部状态"——mood_state 标量、hidden_state 各项数值、overflow 加权分数——都是
**手工设计的规则和公式**,不是从经历里学出来的表示。LLM 本身在两次调用之间完全冻结,唯一
变化的是喂给它的文本和几个人工标量。这是唯一一处"系统没有被经历真正改变"的地方,对应
讨论稿说的"在线学习/快权重"。

### 已经被否决的方案(存档,避免以后重复讨论)

茶茶此前和 Opus 讨论过反向调整角色卡、甚至长期记忆固化后直接去掉角色卡的方案,被否决,
理由是会导致人格漂移/坍缩。这个否决是对的,存档理由:角色卡文本会被 LLM 直接读作身份
声明,是 persona 一致性的锚点,经历直接改写它 = 让锚点本身漂移,没有兜底。

### 可行方案:调"规则参数"而非"人格描述"

这次讨论确认的方向——让经历反向调整 hidden_state 的衰减半衰期、overflow 各项权重、
baseline 目标值这类**元参数**,和上面被否的方案不是一回事:调的是"叶瑄以后怎么感知和
怎么遗忘"，不是"叶瑄是谁"。这些参数不进角色卡、不作为身份声明被 LLM 读取,不会成为
persona drift 的诱因。风险和成本都远低于角色卡级改写或权重级微调,是目前唯一"经历改变
系统结构"里低风险可执行的一档。

具体做法留到排期后再展开,现在只记录方向:storyline 周聚合 / dream 出口的产出,除了写
进 prompt,再开一条支路去调整 `hidden_state_decay` 的衰减参数、`overflow_bucket` 的权重
表、mood_state 的 baseline——经历积累到一定量,系统"感知和遗忘的方式"本身发生变化。

### 终极态(长期研究方向,不排期)

真正的权重级可塑性(LoRA / 定期用 episodic+storyline 微调),对应的正是灾难性遗忘、人格
漂移、安全性问题——这不是设计缺陷,是这个方向本身还在研究前沿。Google 2025 NeurIPS 的
Titans("Learning to Memorize at Test Time")做的是同一类问题:让一个小神经记忆模块在
推理时用梯度下降实时更新,主干权重不动。业界最前沿也才做到"测试时更新一小块记忆网络",
不是"整体人格随经历漂移"的成熟方案。留作长期方向,不必因为没做到就觉得架构有洞。

## 3. World model 部分

讨论稿的自我修正是对的:"限制感知 ≠ 世界模型,真正的世界模型是对不可见世界的预测"。
目前系统对物理环境(Unity 空间、桌上的杯子)完全没有对应实现——这个缺口是诚实的、还没
开始动工的部分,`firmware/presence-device` 和 watch 传感器算是萌芽,离"通过身体经历世界"
还很远。这条依赖 Unity/具身那条线还没启动,不算当前架构的短板,不用现在补。

---

## 4. 产品定位澄清(这次讨论新增,值得记录)

之前一直把叶瑄定位成"陪伴对象",这次讨论明确了一层:茶茶希望的不只是"陪伴",而是**茶茶
不在场时,叶瑄也能存在、变化、成长**——大致可以理解成 `activity_manager` 当前"进阶版"的
非硬编码版本:现在的常驻状态(mood/hidden_state/overflow)已经能在无 owner 交互期间自行
演化(衰减、consolidate、溢出触发主动联系),这个定位澄清确认了这条线是对的方向,而不是
可有可无的锦上添花——以后评估"值不值得做"的时候,这条应该作为优先级判据之一。

## 5. 候选方向汇总(供排期后挑选,当前均不启动)

| 优先级 | 方向 | 成本/风险 |
|---|---|---|
| A(低风险,可先做) | dream/storyline 聚合 → 调整 hidden_state 衰减/overflow 权重/baseline 等元参数 | 低;不碰角色卡,不碰权重 |
| B(长期研究,不排期) | LoRA / 权重级微调 | 高;灾难性遗忘、人格漂移、安全性 |
| 已否决(存档) | 反向调角色卡文本 / 记忆固化后移除角色卡 | 会导致人格漂移坍缩,不再重议 |
| 远期(依赖具身线) | Unity 物理世界模型 | 未启动,依赖单独立项 |
