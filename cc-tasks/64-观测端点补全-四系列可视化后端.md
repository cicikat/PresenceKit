## 1. 端点清单（全部只读 GET，仿 admin/routers/memory.py 模式）

| 端点 | 数据源 | scope | 备注 |

| `GET /growth/works/{interest_id}?limit=` | works/ index.json + 文件（59） | memory.read | 列表 + 单件全文两级 |

## 2. 与前端的契约

Emerald-client 侧面板（SubHiddenStatePanel.tsx 同风格）由**另一张工单**在
Emerald-client/cc-tasks/ 落地（面板五块：成长 / 视觉感知 / 支出 / 群聊仲裁 / 记忆摘要）。

无端点的落盘物视为不可验收。"**

## 4. 测试

1. 每个新端点：有数据返回正确结构、无数据返回空列表、非法参数 422、无 token 401、
   scope 不足 403（复用 test_sec_auth2 模式，default-deny 扫描会自动覆盖新路由）。
2. works 单件读取不越权路径（interest_id 白名单校验，防路径穿越）。
3. `pytest -n auto` 新增测试 + 全路由鉴权盘点守卫回归。

## 5. 不做什么

- 不做任何写端点（观测面只读；写操作各归其主工单）。
- 不做前端（Emerald-client 侧工单负责）。
- 不做 websocket 实时推送（轮询够用，面板不是监控大屏）。
