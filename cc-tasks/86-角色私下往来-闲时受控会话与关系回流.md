# Brief 86 · 角色私下往来：闲时受控会话，只回流关系层（推翻决策 9.5 的受限版）

> 裁决：DESIGN.md §十一 决策 9.5 修订版（原「不做角色间私聊」有条件推翻，理由见该条）。
> 核心：私下往来**可以做**，但产物**只进关系层，不进事实记忆**——这是本单最重要的约束，
> 来自决策 3（自产内容不固化）：私聊全文是模型自产文本，直接 fixation 会造出 owner 从未
> 参与的「事实」并开自我强化回路。关系层（char_relations 本来就是 LLM 生成的印象）是
> 它唯一合法的沉淀地。

## 形态（一句话）

深夜/闲时，调度器低频安排一对角色进行一段 ≤6 轮的短对话；全文落盘仅供管理面板查看，
对角色世界的影响只有三样：char_relations 更新、recent_moments 新梗、presence 一句
「刚才和 X 聊了会儿」。

## 1. 🟡 调度触发（token 硬预算）

- 新 trigger `core/scheduler/triggers/private_exchange.py`：深夜/闲时窗口（同
  `memory_janitor` 时段判断），`stamp_trigger()`，不发言、不进 pipeline。
- 预算：config `private_exchange:` 节——`enabled`（默认 true）、`daily_limit`（默认 1，
  跨所有 pair 合计）、`max_turns`（默认 6，双方各 ≤3）、`window`（默认 deep_night）。
  冷却走 `scheduler_cooldowns.json` 现有机制。
- **一次会话的 LLM 调用数 = max_turns，硬上限断言进测试**（默认配置下每天最多 6 次
  轻量调用，成本量级 ≈ 一轮群聊的零头）。
- pair 选择（纯规则，零 LLM）：候选 = 有 char_relations 记录或共处任一 Stage roster 的
  角色对；加权 = `interaction_count` 低者优先（让新关系有机会长）× 距上次私下往来时间。

## 2. 🟡 会话生成（复用 85 轻量视图）

- 双方轮流生成，走 Brief 85 §1 的 lightweight 视图（角色卡核心 + 双方 char_relations +
  本次往来 transcript，**无长期记忆检索**）。
- 私下语域框定层（system，两条都要）：
  1. 「这段对话{user_name}看不到，不需要表演给任何人，用你们私下的语气。」
  2. **防漂移锚**：「私下语域不等于秘密——你们不讨论对{user_name}隐瞒什么，也不形成
     针对任何人的共识。就是两个熟人闲聊。」
  （第 2 条是硬要求：没有它，「用户看不到」这个 prompt 会诱导合谋叙事，长期经关系层
  回流扭曲群聊人格。）
- 话头素材：双方 activity 动向、recent_moments、节日/时间节点（同 85 §4 引子素材，零新检索）。
- 任意一方生成失败 → 整段放弃、不落盘、不更新关系（fail-open，本日额度不返还）。

## 3. 🟡 落盘与观测（Hard Rule 7）

- transcript 追加 `data/runtime/groups/_private/{char_a}__{char_b}/transcript.jsonl`
  （pair 名按字典序，路径走 `get_paths()` 新方法；滚动保留最近 200 条）。
- 管理面板只读端点：挂 `admin/routers/relations.py` →
  `GET /relations/private-log?char_a=&char_b=&limit=`（scope 同 relations 现有档位）。
  **前端不做任何可视化入口**——想看去管理面板，这是设计（偷看角色私聊的乐趣属于
  管理面板层，不属于日常界面）。

## 4. 🟡 回流（只有三样，白名单式）

1. **char_relations**：会话结束后用**现有** 6h 冷却更新路径喂入本次 transcript
   （不新增调用形态），产出 summary/valence/recent_moments 更新。
2. **presence 提示**：写一个短时 stamp（12h TTL，放 char_relations 同目录小文件或
   activity_state 附注），`2.6_presence` / `2.2_stage_presence` 渲染时可露出
   「刚才和{X}聊了会儿」。不进 event_log。
3. **群聊引子**：85 §4 的引子素材天然包含更新后的 recent_moments，无需额外接线。

**明确禁止**：transcript 及其任何摘要进入 short_term / mid_term / episodic / identity /
event_log / 向量库（负向测试：私下往来跑完后以上各库零新增条目）。角色在与 owner 的
对话中可以自然引用 recent_moments 级别的梗（"前几天和她聊起…"），但没有可召回的
对话全文——就像人只记得聊过什么感觉，复述不了逐字稿。

## 验收

- 默认配置：每日 ≤1 对、≤6 次 LLM 调用（硬预算断言）；enabled=false → 零调用。
- 私下语域两条框定语存在性断言；生成失败 → 零落盘零回流。
- 负向断言：五大记忆库 + event_log + vector_store 零写入。
- char_relations 更新走既有冷却路径；presence stamp 12h 过期。
- 观测端点返回 transcript 尾部；`pytest -n auto`。
- 文档：`docs/stage.md` 边界一节 + `docs/scheduler.md` 触发器表 + DESIGN.md 决策 9.5
  已由本次修订同步。
