# Brief 93 · 首启配置体验:管理面板配置中心 + 密钥本快捷入口 + 通道开关

> 背景:异机真机实测(20260718),新用户走完 AA1→AA2 后仍要手改 config.yaml,
> 且前端裸报 429/401 无指引;QQ 通道默认开着对纯桌宠用户是噪音。
> 本单与 Brief 94 **可并行**;desktop 仓 Brief 34 的引导文案依赖本单 §2/§3 落地后的
> 实际按钮位置/token 名称,先后端后前端。

## 1. 🔴 管理面板「配置中心」页(取代手改 config.yaml)

- 新增 `data-page="setup"` 页,放**第一列最顶部、「创作」组上一行**,命名「配置」。
- 分三层展示,写回 config.yaml(评估复用既有 `settings_llm.py`/`settings_misc.py`
  等 settings_* 路由基建,缺的字段补端点;写回必须走现有配置加载器的校验):
  - **必填(标红,未填时页面顶部横幅警告「未配置将无法聊天」)**:
    基础聊天模型 base_url / api_key / model(即 model_presets 主 preset,
    无 presets 时落 legacy `llm:` 块)。
  - **强烈建议**:embedding(`embedding:` 块,base_url/api_key/model/dim),
    注明作用(长期记忆语义召回)与缺失后果(召回降级,fail-open 不崩)。
  - **可选**:tts、vision、probe/summary 小模型路由(此项只读展示当前
    model_presets roles 指向 + 跳转 model-routing 页的链接,不在本页编辑)。
- 面板首次打开且必填缺失时,自动落在本页(而不是现在的默认页)。
- api_key 输入框按密码框处理,已填时显示掩码 + 「已配置」态。

## 2. 🟡 密钥本快捷入口(面板右下角常驻按钮)

- 面板右下角固定悬浮按钮「打开密钥本」:调后端新端点(admin scope)用系统默认
  程序打开 `secrets.local.yaml`(`os.startfile` / 平台等价物);仅当请求来自
  localhost 时允许,非本机访问按钮隐藏并提示「仅本机可用」。
- 按钮 hover/旁注写明:desktop / mobile 客户端要的 token 在此文件里,
  以及各条目名称与客户端的对应关系(名称以 `scripts/setup_auth.py` 实际写入
  的 key 为准,工单执行时核对)。

## 3. 🟡 AA2 鉴权初始化收尾自动化

- `scripts/setup_auth.py` 成功结束时:自动用系统默认程序打开
  `secrets.local.yaml`,并打开浏览器到管理面板(`http://127.0.0.1:8080`,
  以 config 实际端口为准;服务未启动则改为提示「启动后访问」并跳过)。
- AA2.bat 尾部的静态说明文字同步精简(自动化后不再需要长篇指路)。

## 4. 🟡 通道开关:QQ 默认关 + 面板可视化

- `config.example.yaml`:`qq.enabled` 默认改 `false`,注释写明「需要 QQ 机器人
  再开,并配置 NapCat」;`defaults/` 或代码内置默认值如有同一开关一并对齐。
  确认改动后 standalone 路径零回归(standalone 本来就不走 QQ)。
- auth-tokens 页(`data-page="auth-tokens"`)顶部加「通道开关」区:qq / desktop
  / mobile / device / mail 等各通道 enabled 的可视化 toggle,读写 config 对应
  `enabled` 字段(枚举以 config.example.yaml 实际通道块为准,执行时核对)。
  开关变更需要重启才生效的,toggle 旁标注「重启后生效」,不做热切换(v0.1 不值得)。

## 5. 🟢 probe/summary 小模型兜底确认(已分离,只验兜底)

- 现状:model_presets 已支持 roles(probe/summary 独立指 preset)。本项只做:
  ①核实角色未配置 probe/summary preset 时是否自动兜底主聊天模型——若已兜底,
  文档写明即可;若缺 preset 会报错/阻塞,补 fallback 逻辑(缺失→用主模型,
  记一条 info 日志,不阻塞)。②配置中心 §1 的只读展示接的就是这份真实路由。

## 6. 🟢 后端 401 响应人话化

- admin/ws 鉴权失败的 401 响应体加 `hint` 字段:「token 未配置或已失效,
  请到后端管理面板右下角『打开密钥本』获取对应 token」——desktop 仓 Brief 34
  直接透传显示,文案两边保持一致。

## 验收

- 全新环境:AA1→AA2(自动弹密钥本+面板)→面板配置中心填必填项→desktop 填 token
  →能正常聊天,全程不手开任何 yaml。
- 必填缺失时面板有红色警告且落在配置页;QQ 通道默认关,auth-tokens 页可开。
- probe/summary 缺配置时聊天不阻塞(兜底主模型);`pytest -n auto` smoke 通过。
