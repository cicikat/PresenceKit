## 1. 接口/UI 一致性 lint 测试

`tests/test_admin_mcp_ui.py` 现在的模式（手工列一串字符串断言存在于 `index.html`）
只覆盖 MCP 一个模块。这次 MCP 那轮实际炸出过两个同类问题（`vision-save-msg` 引用
不存在的 DOM 元素、`vision.enabled` 和 `visual_perception.enabled` 两个开关撞脸），
都是"后端概念存在，UI 没对上"的same-shape 问题。

**要做的**：把这个模式推广成一个通用测试——扫描 `admin/routers/*.py` 里所有
`@router.post` / `@router.patch` / `@router.put` 的路径字符串，检查每个是否在
`admin/static/index.html` 里被引用（`api(...)` 调用或字符串字面量）。

**必须有白名单机制**，否则全是误报：一部分接口天生没有 UI（桌宠客户端专用、
纯内部/测试用）。白名单可以是一个显式 set 常量，新增接口默认要求要么出现在 UI
里、要么显式加进白名单并写一行理由——逼着"新增接口忘记接 UI"这件事在 CI 就炸，
而不是等用户点开面板发现少个开关。
