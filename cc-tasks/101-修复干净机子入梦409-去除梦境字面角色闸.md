# Brief 101 · 修复干净机子入梦 409（去除梦境子系统字面角色闸）

> 阻塞 release 开源打包，**插队执行**（先于 Brief 100 / desktop Brief 38）。
> 根因：`dream_pipeline.enter_dream()` 的 fail-closed 闸写死 `char_id != "yexuan"`；
> 干净机子 `character.default: default`（example config）→ 激活角色 `default` → 被拒。
> 私有角色名**不得**为修此 bug 进入任何 tracked 文件（example / 代码 / 文档）。

## 施工项

1. **闸语义改写**：`core/dream/dream_pipeline.py` `enter_dream()`
   `if char_id != "yexuan"` → `if char_id != DEFAULT_CHAR_ID`
   （`from core.data_paths import DEFAULT_CHAR_ID`）。语义：只有部署的默认主角色
   会做梦；多角色部署非默认角色仍 fail-closed（Method B 前语义不变）。
   错误文案维持「这个角色还不会做梦」。
2. **同类字面回退清理**（干净机子会指错目录，不只是观感）：
   - `dream_pipeline._state_char_id()` 的 fallback `"yexuan"` → `DEFAULT_CHAR_ID`
     （log 文案同步去掉字面名）；
   - `_should_retain` 附近 `state.get("char_id") or "yexuan"` → `or DEFAULT_CHAR_ID`；
   - `admin/routers/dream.py` `(... ) or "yexuan"` → `or DEFAULT_CHAR_ID`；
   - `core/data_paths.py` `_read_default_char_id()` 双 fallback `"yexuan"` → `"default"`
     （仅 config 损坏边缘场景生效；与 example 的 `character.default: default` 对齐）。
3. **守卫同步**：`tests/test_no_hardcoded_character.py` allowlist 中被本单消除的条目
   **一并删除**（删除必须连守卫一起删，不留僵尸豁免）；其余条目不动。
4. 顺手 grep `core/dream/` 是否还有其他会影响**运行行为**的 `"yexuan"` 字面量
   （变量名/注释不在本单范围，只清行为分支），有则同法处理并列进执行记录。

## 测试

- 单测：`enter_dream(char_id=DEFAULT_CHAR_ID)` 通过；`enter_dream(char_id="other")` 拒绝。
- 干净机子路径：临时 config `character.default: default` + `characters/default.json`
  下，enter → chat → exit 全链路通（可复用 `docs/fresh-clone-testing.md` 的做法）。
- 存量回归：`character.default` 为自定义值时行为不变（用任意占位 id 测，不写真实名）。
- `pytest -n auto` smoke；allowlist 守卫测试通过。

## 明确不做（另立单）

- tracked 代码中 `yexuan` 作为**标识符/注释**的全量改名（`yexuan_tension` 内部参数、
  `project_body_for_yexuan`、`yexuan_inner_diary` 等约 20+ 文件）——开源零痕迹需求
  另立改名 brief，本单只修行为阻塞项。

## 验收

- 干净克隆 + example config 复制后首次入梦不再 409。
- 本单 diff 中零新增私有角色名字面量；tracked 文件全文 grep 私有名无新增出现。
