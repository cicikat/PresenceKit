# Brief 104 · 新用户冷启动体验（v0.2 对外关键单)

> 背景：v0.1 对外发布后，陌生用户首日体验存在两个断层：
> ① 安装到跑通的引导不足；② identity-2 冷启动期（新用户需
> mid-term → episodic → consolidate 数十轮后才有长期记忆注入），
> 路人前几十轮感受不到记忆深度，易得出"不过是个聊天壳"结论。
> 可并行；§3 依赖 identity-2 的观察数据，允许先落 §1 §2。

## 1. Quickstart 补强

- README（中英）加"10 分钟跑通"路径：装依赖 → 鉴权初始化 → 启动 →
  面板首登 → 发第一条消息，每步给预期输出截图/文案。
- 明确标注最小可玩配置（standalone_mode，无 QQ/NapCat 也能玩）。

## 2. 面板首启检查清单

- 面板检测首次运行（无用户数据）时展示 checklist：LLM key 可用性、
  角色已激活、通道状态；每项给红绿灯和修复指引。
- 复用现有 system-readiness 数据（docs/system-readiness.md），不新造探测。

## 3. 冷启动降级体验

- 目标：identity 未形成前，用已有的 short/mid-term 显式表达"我在慢慢
  认识你"，而非沉默等待固化。方案自定（如 prompt 层在冷启动期加
  轻量提示），但遵守 DESIGN.md 自产内容不固化原则，勿伪造记忆。
- 先量化：记录首个有效 identity 维度平均需要的轮数，作为调参依据
  （对应 known-issues identity-2 的观察项）。

## 验收

- 干净环境按 README 全程跑通不需要读源码。
- 首启面板能一眼看出"哪里没配好"。
- 冷启动期对话中角色不再表现出"完全不记得任何背景"的断裂感。
