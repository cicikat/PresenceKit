# Brief 48 · 查询侧时间意图：解析"上周/那天/N天前"并过滤召回

> 依赖：无（可与 44/45/47 并行）。碰 `pipeline.py::fetch_context` 与 `episodic_memory.py::retrieve` 签名（加可选参，向后兼容）。
> 参考：LongMemEval 时间推理类——写入侧时间处理你已完整（occurred_at / temporal_ref / TTL），
> 但查询侧完全没有：用户问"上周说的那件事"，召回仍按 strength+decay 全时段排序。

## 1. 解析器（新文件 core/memory/temporal_query.py）

```python
def parse_query_time_range(text: str, now: float) -> tuple[float, float] | None:
    """返回 (since_ts, until_ts)；无时间意图返回 None。纯规则，无 LLM。"""
```

- 覆盖：昨天 / 前天 / N天前 / 上周 / 上周末 / 上个月 / 周X（最近一个）/ 具体日期（M月D日）。
- **复用** episodic `event_time_hint` 既有的保守解析工具函数（同一套中文日期逻辑，别写第二份）；
  本地时区，与 episodic 日历日判断一致。
- 保守原则：模糊表述（"之前/很久以前"）返回 None，宁可不过滤也不误过滤。
- 解析失败/异常 → None，fail-open。

## 2. 接线

1. `fetch_context()` 开头调一次 `parse_query_time_range(content, now)`。
2. 非 None 时：
   - `episodic.retrieve(..., since_ts=, until_ts=)` 新增可选参（默认 None=现行为）：
     候选按 `occurred_at`（缺失回退 timestamp）过滤；**时间过滤在关键词/语义候选之后、
     评分之前**；过滤后为空 → 用时间范围内全量记忆参与评分（"上周聊了什么"这类
     time-only 查询没有关键词也要能召回）。
   - vector_store 语义预取传现成的 `since_ts` 参数。
   - `event_log.search` 增加日期范围参数，只扫范围内的日文件（顺带省 IO）。
3. fallback 召回（retrieve_fallback）不接时间过滤（它本来就是近 7 天兜底）。
4. recall_trace 每轮记录 `parsed_time_range`（null 或 [since, until]），供盲区排查。

## 3. 拍板

- 纯规则解析，不加 LLM 探针往返（时间词形封闭集，规则够；漏网词后续按 trace 补规则）。
- 时间过滤只作用于 episodic / event_log / 向量预取三处；lore、profile、diary 层不碰。
- until_ts 上界含当天末尾（"昨天" = [昨日00:00, 昨日24:00)），写测试钉死边界。

## 4. 测试

1. 解析器表驱动用例 ≥ 15 条（含返回 None 的模糊句、跨月/跨年边界、本地时区）。
2. "上周我们聊了什么"：种子含上周与昨天两条记忆 → 只召回上周条目。
3. 无时间词查询 → 与现行为逐字节一致（回归保护）。
4. 时间范围内无任何记忆 → episodic 层为空，不越界兜底（abstention 语义）。
5. 翻转 Brief 44 temporal 类 xfail 用例。

## 5. 不做什么

- 不做"两个事件先后"类相对时序推理（那是回答侧能力，不是召回侧）。
- 不改 format_for_prompt 的时间锚渲染（已完整）。
- 不给 mid_term 加时间过滤（本来只有 12h）。
