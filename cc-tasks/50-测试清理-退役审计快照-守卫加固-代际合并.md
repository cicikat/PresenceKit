# Brief 50 · 测试清理：退役审计快照、守卫加固、代际合并

背景：tests/ 现有 271 个文件约 7.8 万行，其中一批一次性迁移验收快照、文档字符串断言、代际重复测试构成维护负担且存在"误导绿"。本 Brief 分 7 个工单（A–G）。

**并行关系**：A、B、C、F、G 相互独立可并行；D、E 相互独立可并行，但建议在 C 之后做（C 会删掉与 D/E 重叠的文件，先做 C 减少 D/E 的甄别面）。每个工单独立 commit，验收命令统一为 `pytest -n auto` 全绿（部分改动可用 `pytest --testmon`）。

## 工单 A · 修复 test_activity_contract.py 的 silent-skip（P0，误导绿）

现状：`tests/test_activity_contract.py` 通过 `_FRONTEND_ROOT` 跨仓读取 Emerald-client 的 `src-tauri/src/lib.rs` 和 `src/shared/api/activity-api.ts`，文件不存在时 `pytest.skip`。换机器/CI 上永远绿但什么都没测。

改法：
1. 路径解析改为：优先读环境变量 `EMERALD_CLIENT_ROOT`，未设置时回落到现有相对路径推导。
2. 两个路径任一不存在时，默认 **fail**（报错信息里写明设置 `EMERALD_CLIENT_ROOT` 或 `SKIP_CROSS_REPO=1`），仅当显式设置 `SKIP_CROSS_REPO=1` 时才 skip。
3. docstring 说明这是跨仓契约测试及其运行前提。

验收：本机（两仓并存）直接跑为真实执行非 skip；临时改坏路径应 fail 而非 skip。

## 工单 B · conftest 加生产 data/ 写入守卫（P0）

现状：`sandbox` fixture 非 autouse，漏挂它的 I/O 测试会写真实 `data/`。

改法：
1. 在 `tests/conftest.py` 增加一个 **autouse、session 无关** 的默认重定向：每个测试开始时若 `core.sandbox._instance` 未被 `sandbox` fixture 覆盖，则替换为指向 pytest 临时目录的 `DataPaths(mode="test", ...)`。即"默认安全"，`sandbox` fixture 保持现语义（per-test tmp_path，可拿到句柄断言）。
2. 跑全量 `pytest -n auto`，凡因此暴露出"原来在偷读生产 data/"的测试，逐个改为显式使用 `sandbox` 并自备 fixture 数据（这类每修一个单独 commit）。
3. 注意 config.yaml 在仓库根、不在 data/ 下，不受影响；不确定时读 `core/sandbox.get_paths()` 实现确认边界。

验收：全绿，且手工在守卫中放一个故意写 data/ 的临时测试能被隔离（不落盘到真实 data/）。
