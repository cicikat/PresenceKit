# Brief 46 · 遗忘=降级而非删除：淘汰压缩 + event_log 过期抢救

> 依赖：**45**（抢救产出的事实要走 ADD/UPDATE/NOOP 入口，否则重复堆积）。
> 与 47/49 同碰 `episodic_memory.py`，合并时注意串行。
> 参考：Learning to Forget（arXiv 2604.11306）降级式遗忘；RecMem（arXiv 2605.16045）回访原始交互抢救持久事实。
> 现状问题：episodic 满 200 条直接删最低 strength 的 20 条；event_log 30 天过期即消失。
> 低情绪但持久的信息（计划、承诺、生活变化）只有 eager（sad/angry/happy）和 sweep 两条晋升路，漏了就永久丢。

## 1. episodic 淘汰改"压缩归档"（core/memory/episodic_memory.py + fixation_pipeline.py）

`write_episode` 触发上限裁剪时：

1. 被裁的 20 条**先入队** slow_queue 新任务 `digest_evicted_episodes`（payload 携带条目全文 + char_id 快照），再从 episodic.json 删除（现行为）。
2. handler：一次 LLM 调用把 20 条压成 5–8 行"时期摘要"（保留时间跨度、反复出现的主题、
   用户状态变化），追加写入 `data/runtime/memory/{char_id}/{uid}/memory_digest.md`
   （路径经 path_resolver 注册新 artifact key `"memory_digest"`，Hard Rule 1）。
3. 每次追加带日期头 + 来源 ep_id 列表（血缘可追溯）；`provenance_log.append(artifact="episodic", trigger_signal="evict_digest")`（Hard Rule 6）。
4. LLM 失败 → 原文条目以紧凑 JSON 追加到同文件的 `<!-- raw -->` 区块，不丢数据，下次任务不重试（fail-open）。

## 2. event_log 过期前抢救 pass（core/scheduler/triggers/ 新增 event_log_salvage.py）

调度器每日触发（模式仿 `hidden_state_decay.py`：`stamp_trigger()`，不发言）：

1. 扫描 age ∈ [27, 29] 天、尚未抢救的 event_log 日文件（状态标记见拍板）。
2. 每个文件一次 LLM 调用：提取"仍然为真的持久事实"（偏好、身份信息、生活状态、承诺），
   显式排除一次性事件和情绪表达。
3. 产出走 **Brief 45 的 `extract_and_update` 冲突裁决入口**（op=add/update/noop），
   不新建存储——已被 profile/episodic 覆盖的信息会被 noop 掉。
4. 每日处理上限 3 个文件（防积压时一次打爆 LLM 配额），冷却由 scheduler 现有机制管。

## 3. 拍板

- **memory_digest.md v1 不注入 prompt**：先只归档，观察内容质量 2–4 周后再决定是否作为
  低优先级层（届时按 Hard Rule 3 加 `_layer`）。本 brief 不碰 prompt_builder。
- 抢救状态标记：`fixation_state.json` 新增 `salvaged_dates: [YYYY-MM-DD]`（滚动保留 60 个），
  不新建状态文件。
- 两个 LLM pass 均为 `WriteEnvelope` 门控写入；`is_test` 强制不可写（现有契约自动覆盖）。

## 4. 测试

1. 上限裁剪 → digest 任务入队且 payload 完整；episodic.json 结果与现行为一致；核心记忆仍不参与裁剪。
2. digest handler：LLM mock 成功 → md 追加带血缘头；mock 失败 → raw 区块兜底，无异常。
3. salvage：age 28 天文件被处理且 `salvaged_dates` 记录；再跑一遍跳过（幂等）；age 10 天不处理；单日 >3 个到期文件只处理 3 个。
4. 抢救产物经 45 的 noop 去重：种子 profile 已含同义事实时 facts 数量不增。
5. `pytest -n auto` 指定新增测试文件 + `tests/test_fixation_pipeline.py`。

## 5. 不做什么

- 不做 mid_term 12h 过期的抢救（有 sweep 晋升兜底，先不叠）。
- 不给 digest 建向量索引、不进召回路径。
- 不改 200 上限本身。
