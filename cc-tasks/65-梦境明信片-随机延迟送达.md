# 工单 65：梦境明信片 —— 合格梦境生成书信，随机延迟 1–356 天送达邮箱

> 由 Claude Desktop 与用户聊定方案后产出，Claude Code 执行。
> 与 66 / 67 / 68 **无依赖，可并行**（68 的"明信片暗线"子任务软依赖本单，见 68）。
> 改动前必读：`docs/dream.md`（合同段）、`docs/scheduler.md`。

## 背景与定位

梦境结束后，以叶瑄口吻给用户写一封关于这场梦的短信/明信片，经现有 Gmail 发信链路
（周信所用的那条）投递，**送达时间随机延迟 1–356 天**——用户可能在一年后翻邮箱
"抽到"一封当年的梦境信。

**隔离定性（写进代码注释与 docs/dream.md）**：明信片是 archive 层的出站复盘形式
（archive 定位="仅她复盘，任何 loader 永不读"）。方向是 梦→用户的眼睛，单向出站：
- 读：dream summary + archive 原文片段（这是 archive 第一个合法程序化读者，仅限本模块）
- 写：仅 postcards/ 调度文件 + SMTP 发信
- **绝不写任何 memory / mood / hidden_state / impression**；不进任何 prompt loader
- 世界专有词**允许出现**（用户 facing，不走 strip_vocab——这不是回流）

## 触发条件（全部满足才生成）

1. `dream_mode == "sandbox"`（scenario / mirror 不发信）
2. 本场有效 assistant 轮数 ≥ 5（统计方式对齐 `RETAIN_MIN_TURNS` 的轮数口径）
3. `exit_type != "hard_exit"`（被 Esc 掐断的梦没有结语，不配信）
4. `dream_id` 未发过信（去重）

**没有掷骰、没有冷却期**——合格就写，一场一封。随机性全部放在两处：
模板选择 + 送达延迟。

## 实现

### 1. 生成（挂在 `_generate_summary_bg()` 完成之后，fail-open）

- 新模块 `core/dream/postcard.py`。
- 前置 prompt **多模板随机**：`characters/dream_postcards/templates/*.md`，本单先写 5 个
  不同文体（明信片 / 短信 / 日记撕页 / 便签 / 无题短笺），生成时随机选一，记录 `template_id`。
- 输入：summary.json + archive 末尾若干轮片段。
- **信内日期必须写做梦当时的时间，精确到时分**（取入梦时间戳，格式如
  `2026-07-11 23:47`），**绝不写送达时间**。
- 生成失败 → warning，不阻断 dream exit（与 summary 同策略）。
- 信文本在**生成时冻结**存盘（不要送达时才现生成——archive 格式将来可能变）。

### 2. 调度存储

- `data/runtime/dreams/{char_id}/postcards/schedule.json`，路径**必须走
  `core/sandbox.get_paths()`**（Hard Rule 1）。
- 条目字段：`dream_id / uid / dream_time_iso / template_id / letter_text /
  scheduled_date / sent(bool) / attempts / last_error`。
- `scheduled_date` = 生成日 + uniform(1, 356) 天；**按"日"粒度查重**，与已有未发送
  条目撞日则重掷（最多重掷 10 次，仍撞则顺延一天）。

### 3. 投递（scheduler proposer）

- 新 proposer：每日检查 schedule.json 中 `scheduled_date <= today && !sent` 的条目。
- 复用现有 Gmail 发信链路发出；成功置 `sent=true`。
- **发送失败（网络/梯子/SMTP 抽风）→ 不丢信**：`attempts+1`、记 `last_error`、
  下一 tick 重试，直到成功。这是用户明确要求（"挂着梯子信被网卡住"场景）。

## 测试

- 触发门槛正反例：hard_exit 不生成 / 4 轮不生成 / scenario 不生成 / 合格 sandbox 生成
- 撞日重掷；发送失败重试不丢条目
- **隔离合同（反假绿铁律，正反都要）**：断言 postcard 模块无任何 memory 写路径；
  正样本断言它确实读了 archive（证明扫描有效，不是空集断言）
