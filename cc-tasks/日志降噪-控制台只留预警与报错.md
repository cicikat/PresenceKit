# 工单：后端控制台日志降噪（只留 WARNING / ERROR，噪音进文件不进屏）

> 仓库 `D:\ai\qq-st-bot`。扩展点：`admin/log_filter.py`（已有 access 脱敏 + asyncio 噪音过滤的同款 Filter 模式）。
> 目标：控制台只留预警和报错，避免真报错被 INFO 刷屏淹没。**文件日志保持完整**，只让控制台安静。

---

## 背景

控制台现在被两类高频 INFO 刷屏：
1. uvicorn 访问日志：`INFO:  127.0.0.1:53833 - "GET /lorebook HTTP/1.1" 200 OK` —— 每个面板请求一条，纯噪音。
2. `prompt_builder.debug` 的 `[layer_size] 1_system_prompt: 409 chars`（`core/prompt_builder.py:1108-1113`）—— 每轮每层一条；这信息已迁进 round4 的 Prompt 层检视器，控制台无需再印。

这些把真正的 WARNING/ERROR 淹没。

---

## 改法（沿用 `log_filter.py` 现有 Filter 模式）

1. **uvicorn 访问日志：丢掉成功请求（2xx/3xx），保留 4xx/5xx。**
   - 在 `log_filter.py` 加 `DropSuccessfulAccessFilter(logging.Filter)`：uvicorn.access 记录的 `record.args[4]` 是 status_code，`200<=code<400` 时 `return False`（丢弃），其余放行。这样 404/500 这种仍会出现在控制台。
   - 加 `install_access_noise_filter()`，对 `logging.getLogger("uvicorn.access")` `addFilter`。
   - 与现有 `QuerySanitizeFilter` 并存（脱敏仍对保留下来的 4xx/5xx 生效）。

2. **`prompt_builder.debug`（layer_size）：抬到 WARNING，INFO 不再上屏。**
   - 启动时 `logging.getLogger("prompt_builder.debug").setLevel(logging.WARNING)`。
   - **注意区分**：`prompt_builder.token` 是另一个 logger，裁剪/超预算用的是它的 `.warning()`（`prompt_builder.py:1076/1100/1105`）——**这些是有用预警，保留**。只压 `prompt_builder.debug`，别误伤 `prompt_builder.token`。

3. **总开关（可选但推荐）：config 控制控制台 verbosity。**
   - 加 `config.yaml`：
     ```yaml
     logging:
       console_quiet: true     # true=只 WARNING+ 上屏 + 丢成功 access；false=恢复啰嗦，深度调试用
     ```
   - `console_quiet: true` 时：装上述两个过滤器 + 把控制台 handler 级别设为 WARNING（或对几个高频 INFO logger 单独抬级，避免一刀切吞掉启动信息——二选一，倾向"抬高频 logger + 丢成功 access"，保留启动/关键 INFO）。
   - `false` 时一切照旧。默认 `true`。

4. **文件日志不动**：`/logs` 端点读的错误日志文件（`system.py` get_logs）、`forensic_logs`、`prompt_builder.token` 文件输出等保持完整级别，**降噪只作用于控制台 handler**。确认过滤器只挂控制台 handler / 对应 logger 的 propagation，不影响落盘。

5. **安装时机**：与现有 `install_access_log_sanitizer()` 同处调用（`admin/admin_server.py:107-109` 的 `start_admin_server`，以及 main 启动路径）。把新 install 函数一并在那里调用；确认 `install_asyncio_proactor_noise_filter()` 也已被调用（若漏调顺手补上）。

---

## 验收

- 正常面板操作（GET /lorebook 等 2xx）控制台**不再刷访问日志**；故意打一个 404 → 控制台**仍能看到**。
- 对话若干轮，控制台**不再出现** `[layer_size] ... chars`；但制造一次 >20k 触发裁剪 → `prompt_builder.token` 的裁剪 WARNING **照常出现**。
- 文件日志 / `/logs` 端点内容不受影响，仍完整。
- `config.yaml` 把 `logging.console_quiet` 改 `false` 并重启 → 啰嗦日志恢复。

> 改完手动起一次 `python main.py`，跑几个面板请求 + 发几条消息，肉眼确认控制台只剩预警/报错与关键启动信息。
