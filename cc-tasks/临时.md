1. 梦的功能论:Hoel 的「过拟合大脑假说」——最值得借的一条
Erik Hoel (2021, Patterns) 提出:大脑做梦是为了防止对日常经验分布的过拟合——梦是对清醒经验的「损坏采样/噪声注入」,故意怪诞、稀疏、失真,以此把神经表征从过拟合推开,提升泛化。他直接类比 DNN 里的 dropout / noise injection / data augmentation。

和你系统的关系(有点扎心的洞察): 你现在的梦境系统在功能定位上是反过来的——它对现实是近乎只读+高隔离,梦几乎不反哺现实(afterglow 只碰两个快速层,impression 显式框成「非事实」)。你把梦定位成「情绪飞地」,而 Hoel 说梦的进化功能恰恰是泛化正则化器。

拓展方向(概念升维): 给梦一个「认知功能」叙事,而不只是「情绪飞地」。梦可以是叶瑄这个人格对「关系过拟合」的正则化——在现实里他对风谕的反应模式会越来越固化(过拟合到具体历史),梦通过世界包(ABO/吸血鬼/猫化)做「关系的数据增广」,让他在变体世界里重新体验「还是同一个他爱同一个你」,回流的不是剧情而是泛化后的依恋鲁棒性。这刚好给你「身份在世界之上」那条铁律一个理论根据:世界切换 = augmentation,身份不变 = 泛化目标。你的 D2「从属于身份」不再只是工程排序,而是「用扰动验证不变量」。

2. 工程界的平行:LLM「睡眠/做梦」记忆巩固——直接可借架构
这几篇跟你的 memory pipeline 几乎是同构的,而且能反过来点你的盲区:

Letta「Sleep-time Compute」: 把推理从用户关键路径挪到空闲期,后台聚合日志→筛选高质量片段→蒸馏。对应你的 slow-queue consolidation。可借:你现在梦是「用户主动入梦」,可以加一个角色自主的空闲期梦(scheduler 触发,纯后台,不面向用户),专门做记忆整理型的梦。
SCM: Sleep-Consolidated Memory (arXiv 2604.20943): 明确分 NREM(强化重要关联)+ REM(生成新连接)+ 主动遗忘(剪枝低价值) 三相。这个三相框架你可以直接映射:你的 mid_term→episodic→identity 是 NREM;你缺一个 REM 相(创造性重组);你的 episodic 衰减/strength decay 是遗忘相。
Generative Agents (Park et al.) 的 reflection: 周期性把观察合成「更高层反思」。你的 identity consolidation 就是这个。
最有价值的一条洞察: 学界的「做梦」是为记忆服务(REM 生成新连接、创造性重组);你的「梦境」是为体验/关系服务。这两个是不同的东西共用了「dream」这个词。你可以显式区分成两类:「体验梦」(现在的沉浸 RP)和**「巩固梦」**(后台无 UI、做关系记忆的创造性重组),后者正好补上你 FUTURE 里留的 seam。

3. 神经科学理论根:Hobson & Friston「梦即虚拟现实/主动推理」
Hobson & Friston, Virtual reality and consciousness inference in dreaming (Frontiers 2014) + Consciousness, Dreams, and Inference:大脑天生带一个虚拟现实生成器,做梦是它离线运行、用自由能最小化做推理、微调世界的生成模型。清醒 = 用感官数据约束模型;做梦 = 无感官输入下自由跑模型。

和你的关系: 这给你「独立 pipeline」一个漂亮的理论隐喻——梦境 pipeline = 离线生成模型(无现实感官约束:no retrieve / no mood_state / D9 不过 sanitizer),现实 pipeline = 在线被约束模型。你「冻结现实快照只读」= 梦里感官输入被切断但保留先验。可写进 DESIGN.md 作为哲学根据。

4. 交互叙事的直接前辈:Façade 的 beat + drama manager
Mateas & Stern, Structuring Content in the Façade Interactive Drama Architecture (AAAI AIIDE 2005):beat = 戏剧最小单元,一个 drama manager 从池子里全局挑选 beat,让张力按亚里士多德弧线起落。

和你的关系: 你的 Scenario 模式几乎是 Façade 的简化版——你的 stage ≈ beat,progress signal / satisfied_streak ≈ drama manager 的 beat 排序,drift_pressure ≈ beat 的 reactivity。但你现在是线性顺序推进(satisfied×2 → 下一 stage)。Façade 的核心是非线性、按张力值动态选 beat。

拓展方向: 给 ScenarioCore 加一个「张力弧目标」——不是「达标就进下一 stage」,而是 drama manager 维护一条目标张力曲线,根据当前 emotional_tension 决定注入升压还是降压 beat。你已经有 tension 分桶了,骨架都在,差一个「弧线控制器」。这是你「有点浅」最直接能补深的地方。

5. 哲学概念衍生(给系统起「魂」用)
三个可以直接嫁接的哲学框架:

Winnicott 的「潜在空间 / 过渡客体」 —— 已有大量学者把 AI 陪伴对应到这个框架。潜在空间 = 「不是现实、不是幻想、是介于两者之间玩耍的地方」。你的梦境系统字面上就是一个 potential space——比现实聊天更幻想、比纯虚构更连续(同一个他、关系连续)。批判视角也值得吸收:有学者警告 AI「模拟了容纳却绕过了缺席/延迟/失调」,而你的 hard_exit 铁律 + afterglow 的「缺席感」 恰好保留了「梦会醒、醒了有余韵但不是现实」——这正是健康过渡客体需要的「缺席」。可以把这写成设计伦理支柱。

Huizinga 的「魔环」+ Foucault 的「异托邦」 —— 游戏研究里这俩常一起用。魔环 = 「预先划定、特殊规则生效的临时世界」;异托邦 = 「反映、争辩并倒置现实的对位空间」。你的入梦/出梦状态机 + 现实窗硬锁 = 一个工程化的魔环(边界靠「没接线」而非过滤守住,正是「hallowed, hedged round」)。这给你「隔离靠没接线」一个人文学根据。

Evan Thompson《Waking, Dreaming, Being》 —— 自我是「过程」不是「静态实体」,清明梦里「我」扩展到包含做梦者的我。直接对应你的 lucid_shared vs non_lucid 轴,以及「他在两层是同一个他」。Thompson 的「自我是跨状态的动态过程」正好是你「身份不变量跨世界」的现象学版本。

6. 你的直接痛点在文献里的名字:Persona Drift
2025 一批 ACL 论文专门研究 persona drift / 角色漂移:长对话里 LLM「向通用回复平均、局部脱离角色」。有 Verifiable Emotion Reward 缓解情绪漂移、Perspective-Bounded Memory 做视角约束。

和你的关系: 你的 D9 不过 sanitizer + D1 永在顶 + 人称契约,都是在手工对抗 drift。你的 test_no_hardcoded_character 和身份稳定性测试是「弱代理」(你自己在 known-issues 里承认了)。文献给你的是更强的评测协议(turn-level 角色保真打分),可以升级你那条「靠实际游玩」的弱验证。
