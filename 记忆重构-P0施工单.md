## 命名约定（四项共用，先读）

新增两个语义，**不要复用现有字段**：

| 字段 | 语义 | 备注 |
|---|---|---|
| `timestamp`（已存在） | **recorded_at**：这条记忆被反思/写入的时刻 | 保持不动；decay / index / MMR / 排序继续用它 |
| `occurred_at`（**新增**） | 事件**真实发生**时刻的最佳估计 | 渲染时间锚用它；缺失时回退 `timestamp` |
| `event_time`（已存在） | **未来**事件的预定时刻（TTL/expiry 用） | 语义不变，**勿与 occurred_at 混用** |

## P0-1 召回不缝合、按说话人成"第三人称事实卡"（治 bug 1 显性面）

**文件**：`core/memory/event_log.py` → `search()`（行 306-373）
**目标**：`search()` 输出不再是跨说话人 `; ` 缝合 + 80 字裸截断的乱串，而是**每条带明确归属、整句、第三人称**的事实卡。`prompt_builder` 层 6b 无需改（继续注入返回的字符串），但内容已无歧义。

### 改造逻辑

1. 块内逐行扫描时，**追踪当前说话人**：
   - 行以 `**用户**：` 开头 → role=`user`，剥前缀取正文；
   - 行以 `**{char_name}**：` 开头 → role=`assistant`，剥前缀取正文；
   - **无 `**…**：` 前缀的续行** → 继承上一条 role（防止 assistant 正文里的内嵌行被误判，配合 P0-2）。
2. 命中判定（关键词）对**剥前缀后的正文**做，不对前缀做。
3. 命中的行**单独成卡**，不跨说话人 `join`。正文按**句末**截断（。！？；或长度上限 60，取最近句号），不做裸 `[:80]`。
4. 渲染成第三人称卡，时间用块的 `days_ago` 粗粒度（event_log 本就按天，永不出现"几小时前"）：

```
（{coarse}）你提到：{text}
（{coarse}）{char_name}当时说：{text}
```
`coarse`：`days_ago==0→"今天"`、`==1→"昨天"`、`<7→"前几天"`、`<30→f"约{days_ago}天前"`。

5. 返回值结构不变（仍 `str` 或 `(str, trace)`），`result_str` 改为 `"\n".join(cards)`；`trace` 每项加 `role`、`event_day`。

### 代码骨架（替换 search 内 348-366 的聚合段）

```python
_ROLE_PREFIX_RE = re.compile(rf"^\*\*(用户|{re.escape(char_name)})\*\*[:：](.*)$")
# char_name 来自 from core.config_loader import _char_name; char_name=_char_name()

cards = []  # (final_score, role, text)
for block in _split_blocks("\n".join(lines[1:])):
    intensity = _parse_intensity(block)
    if days_ago > 7 and intensity < 1:
        continue
    score = intensity * decay
    cur_role = "assistant"  # 块首默认；遇到前缀即纠正
    for line in block:
        s = line.strip()
        if not s or s.startswith("#") or s == "---" or s.startswith("> emotion:"):
            continue
        m = _ROLE_PREFIX_RE.match(s)
        if m:
            cur_role = "user" if m.group(1) == "用户" else "assistant"
            body = m.group(2).strip()
        else:
            body = s  # 续行，继承 cur_role
        if not body:
            continue
        hit = sum(1 for kw in keywords if kw in body)
        if hit > 0:
            text = _clip_sentence(body, limit=60)
            cards.append((score + hit / max(len(keywords), 1), cur_role, text, days_ago))

cards.sort(key=lambda x: x[0], reverse=True)
selected = [c for c in cards[:5] if c[0] >= MIN_SCORE]   # MIN_SCORE=0.6 不变
def _render(role, text, days_ago):
    coarse = ("今天" if days_ago==0 else "昨天" if days_ago==1
              else "前几天" if days_ago<7 else f"约{days_ago}天前")
    who = "你提到" if role == "user" else f"{char_name}当时说"
    return f"（{coarse}）{who}：{text}"
result_str = "\n".join(_render(r, t, d) for _, r, t, d in selected)
```

`_clip_sentence(body, limit)`：在 limit 内取最后一个句末标点（。！？；）截断；无标点则硬截到 limit 并加"…"。新增小工具函数即可。

### 单测（新增 `tests/test_event_log_search_cards.py`）

- 用户行 + 角色行混在一个块 → 返回两张卡，role 各自正确，无跨说话人拼接。
- 角色块正文含内嵌无前缀行（如 `用户：我没有这么写`）→ 该行**继承 assistant**，不得渲染成"你提到"。
- 60 字长正文 → 句末截断，不在词中间断。
- `days_ago` 渲染映射正确（0/1/3/10）。
- 既有 `MIN_SCORE`、最多 5 条、30 天窗口行为不回归。

### A. 写入：派生 occurred_at

新增两个 helper（模块级）：

```python
def _parse_turn_ms(turn_id: str) -> float | None:
    """turn_id 形如 {uid}_{ms}；取尾段毫秒→秒。"""
    try:
        return int(str(turn_id).rsplit("_", 1)[1]) / 1000.0
    except (IndexError, ValueError):
        return None

def _derive_occurred_at(to_process: list[dict], fallback: float) -> float:
    """取这批 mid_term 来源 turn 的最早真实时刻；都拿不到则回退（=反思时刻）。"""
    times = []
    for e in to_process:
        ms = _parse_turn_ms(e.get("source_turn_id") or "")
        if ms is not None:
            times.append(ms)
        elif isinstance(e.get("ts"), (int, float)):
            times.append(e["ts"])
    return min(times) if times else fallback
```

## P0-4 灭回忆永动机 + 去陈旧优先权（治 bug 2 复发）

**文件**：`core/memory/episodic_memory.py` → `retrieve_fallback()`（行 628-681）+ `retrieve()` 强化段（行 469-492）

### A. fallback 用 occurred_at 卡窗口，且核心记忆不靠"近期高强度"无条件浮起

`retrieve_fallback` 行 645-648：

```python
anchor = m.get("occurred_at")
if not isinstance(anchor, (int, float)):
    anchor = m.get("timestamp", now)
age_days = (now - anchor) / 86400
if age_days > 7:
    continue
if m.get("is_core") and age_days > 2:
    # 核心记忆不靠"近期高强度兜底"反复复活；仅在真正近 2 天内才允许兜底浮起
```

> 这样：被重新晋升的"生日"回顾型 episode，即便 strength 高，只要其真实 `occurred_at` 不在近 2 天，就不会从 fallback 反复冒头。配合 P0-3，即使冒头也只会渲染"之前"。

### B. 召回强化加冷却（防 strength 单调爬升）

> 现状：`fetch_context`（读路径）已传 `allow_strengthen=False`（pipeline.py:243-244 注释 + N2-A），读路径不写回，永动机已基本关闭。本项是**对写路径的防御性硬化**，优先级低，可选。

`retrieve()` 行 470-481 强化段，加冷却 + 上限：

```python
COOLDOWN_S = 6 * 3600
STRENGTH_CEIL_NONCORE = 0.9
for mem in memories:
```
