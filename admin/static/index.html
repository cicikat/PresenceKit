⚠ 未配置将无法聊天/主动触发失效：请先填写下方「基础聊天模型」必填项

基础聊天模型 必填①

日常对话实际调用的模型连接。已配置 model_presets 时写入当前生效路由的主聊天 preset；未配置时写入旧版 llm: 块——两种情况都会自动识别，不需要你判断。更细粒度的多模型 preset / 路由切换见「模型路由」页。

💡 当前用的还是内置占位角色卡（"角色名"），去「角色卡」页创建/编辑属于你自己的角色，聊天才会有真实人设。

Owner ID 必填②

建议直接填你的 QQ 号——若此处用了别的 id，之后接 QQ 时会按 QQ 号另起一套记忆，与桌宠期记忆不互通。留空 = 主动触发调度器会静默跳过主动消息，且记忆文件可能挂在错误/缺省 uid 下。

语义 Embedding 强烈建议

作用：长期记忆语义召回（情景记忆/往事检索按语义匹配而非纯关键词）。缺失后果：召回质量降级为关键词匹配，不会导致聊天崩溃或阻塞（fail-open）。

模型路由

这个页面是干什么的？

三个概念，从下往上理解：
① Preset（模型连接）= 一个"你能用的模型"：服务商地址 + 密钥 + 模型名， 填一个 preset 相当于告诉后端"这里有一个能调用的模型"。
② Routing Profile（分配方案）= 一套"谁用谁"的表：把不同用途的调用 （比如角色正式回复 vs. 后台判断/摘要这类不需要用户看到的杂活）分别指给某个 preset。 不是每类都要填——留空的类别会自动回退用这套方案里的 chat， 所以新手只填 chat 一项就能跑起来。
③ 当前生效路由方案 = 现在实际在用的那一套分配方案。可以配多套（比如「日常省钱」 「重要场合上强模型」），需要时切换。

配置顺序：先在下方「Presets」新建至少一个模型连接 → 再到「Routing Profiles」新建/编辑一套方案， 把 chat 指向刚才建的 preset → 最上方「当前生效路由方案」 切换到这套方案即可生效。

当前配置未开启 model_presets 块（legacy 模式，等价合成了单个 legacy preset）。请先在 config.yaml 里加上 model_presets: 顶层块（参见 config.example.yaml），可点击右侧按钮把当前 legacy 连接初始化为可编辑 preset。

当前生效路由方案

下拉框选一个下方已建好的 routing profile，点「切换」立即对全局生效。 注：如果某个角色卡自己单独绑定了一套方案（在「角色卡」页设置），该角色对话时优先用它自己绑定的那套， 不受这里的全局选择影响。

Presets

一个 preset 就是一个"模型连接"：填好服务商地址、密钥、模型名，就能在下方 Routing Profiles 里被指派任务。 新建后建议先点「测试」，确认这组配置真的能打通再拿去用。

Routing Profiles

一个 profile 就是一套"把哪类调用交给哪个 preset"的分配表（call_category → preset）。 没单独指定的类别会自动回退用这套方案里的 chat， 所以只填 chat 也能正常跑；其余类别都是不面向用户的后台 "杂活"，想省钱可以单独指到便宜模型。

各 category 是干什么的？点开看
chat —— 角色的正式回复，用户实际看到的每一句话。质量要求最高，建议配主力模型。
intent —— 判断"要不要触发某个动作"的轻量辅助判断，不面向用户，配便宜模型即可。
probe —— 每轮先判断要不要调用工具的轻量"探针"调用，配便宜模型即可。
summary —— 生成摘要（压缩长对话、总结一局陪玩/一场梦），配便宜模型即可。
detect_emotion —— 识别文字里的情绪标签，配便宜模型即可。
consolidation —— 把短期记忆整理沉淀为长期记忆、提炼关系事实等后台整理，配便宜模型即可。
perform —— 把回复文字映射成动作/表情演出指令（只有开启"表演意图映射"功能时才会用到）。
monologue —— 角色说话前的内心独白/思考草稿（只有开启"思考链"功能时才会用到）。
