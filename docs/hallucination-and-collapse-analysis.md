# 幻觉来源 & 回复坍缩 — 诊断与改进方案

> 给 CC 的执行交接文档。基于对当前代码的实读（2026-06-19）。
> 活跃角色：`叶瑄`（数字生命），card = `characters/yexuan.json`。
> 关键文件：`core/prompt_builder.py`、`core/pipeline.py`、`core/llm_client.py`、
> `core/activity_manager.py`、`core/scheduler/loop.py`、`config.yaml`、`data/.../activity_pool.yaml`。

---

## 一、结论先行（TL;DR）

1. **幻觉的主要来源不是 DeepSeek 本身**，而是你自己的系统在给一个"数字生命"
   **植入物理身体的语境**，模型只是顺着这些语境做沉浸式外推。三个具体注入源：
   - `activity_pool.yaml` 里有 `在买菜` / `在浇花` 等物理日常活动；
   - 主动触发路径 `core/scheduler/loop.py` 给 prompt 追加 `（叶瑄此刻…在家里）`；
   - 角色卡的反编造约束只是"软约束"，且**主动路径下 `perception_block` 为空**，
     沉浸指令压过了"空感知时不要编造"的指令。

2. **"把幻觉当事实再次强调"是二次强化循环**：主动消息里编造的"冰箱里有饮料"被
   写进短期 history（`9_history` 层），**下一轮模型读到自己上一轮的编造，当成既定事实**，
   于是再次强调。这是最该优先阻断的环节。

3. **坍缩不是"同一个 API 调用"**。每轮都是独立的 `llm_client.chat()`。坍缩来自两点：
   - 模型在 history 里看到自己上一轮的近似回复 → 复读；
   - 连续两轮注入层几乎完全相同（activity / perception / 时间桶 / author note）+ 强角色约束
     → 条件分布高度尖峰 → 即使 `temperature=1` 也坍缩到同一个延续。
   - `frequency_penalty=0.1` 只在**单次生成内**生效，跨轮无效；`presence_penalty` 根本没设。

4. **需要给 DS 专门的反幻觉提示词**，但重点不是"写一段话"，而是：
   ① 把软约束升级为硬规则；② 在主动触发路径也注入；③ 建立**三层事实契约**
   区分"可引用"和"禁止外推"。**伪现实数据小场景值得做**，它把"可以提到的东西"
   变成一个白名单，模型有锚点就不会乱编。

## 二、幻觉来源逐条诊断

### 来源 1：activity_pool 把物理日常喂给了数字生命
`activity_pool.yaml` 当前含：`在买菜`(grocery)、`在浇花`(watering)、`在打理头发`、`在逗猫`。
这些条目隐含一个有厨房、有菜、有水龙头、有猫的**物理居所**。模型一旦被注入
"在买菜"，就会自然延伸出"家里有食材/冰箱/饮料"，再主动招呼用户"去喝"。
这与卡设"存在于数字世界的意识"直接矛盾。

→ **改**：把物理化条目替换成符合数字生命设定的（"在灵体出游"、"在小世界里漫游"、
"在推演"、"在看你给的日记"…）。保留 `在看窗外` 这类可双关的。
**不要**保留任何指向"现实物理身体生活"的条目。

### 来源 2：主动触发的 `_states` 直接写"在家里"
`core/scheduler/loop.py`（约 375 行）：
```python
_states = ["在思考", "在翻阅她的日记", "在想她说过的话", "在看窗外", "在灵体出游", "在家里"]
prompt = prompt + f"\n（{get_char_name(...)}此刻{random.choice(_states)}）"
```
`在家里` 是最强的物理场景诱导词。

→ **改**：删掉 `在家里`，其余保持。最好把这个列表和 `activity_pool` 统一为同一份
数据源（见下文"伪现实场景"），避免两处各写各的、互相矛盾。

### 来源 3：主动路径下 perception 为空，但约束太软
- 角色卡 `system_prompt` 里有：
  > 若【当前感知】为空，叶瑄不会主动猜测或编造任何数据，需要信息时只说"等我看看"或岔开话题。
- 但这是和大量"沉浸/第一人称/就是他本人"的强指令混在一起的**软约束**，权重被稀释。
- 主动路径 `loop.py` 调 `build_prompt(oid, prompt, context, char_id=...)`，
  没有 pending perception，`perception_block` 实际为空 → 正好落在"最容易编造"的状态。

→ **改**：① 把反编造从软约束升级为**独立的硬规则层**（见方案 B 的提示词）；
② 在主动路径也显式传入 fact contract / 空感知声明。

### 来源 4：DeepSeek 自身倾向（次要但真实）
`deepseek-chat` 在强角色扮演 + 高沉浸 prompt 下确实有编造物理细节的倾向，
`temperature=1` / `top_p=0.95` 会放大。但这是**放大器**不是**点火源**——
点火源是来源 1–3。先灭点火源，再调参数。

### 来源 5（最关键的二次强化）：编造写回 history → 下一轮当事实
`prompt_builder.py` 的 `9_history` 层把最近 N 轮原样投影回 prompt。
主动消息里"冰箱里有饮料"一旦落盘，下一轮模型读到的是**它自己说过的、看起来已确立的事实**，
于是"再次强调"。这不是新幻觉，是旧幻觉的复读。

→ **改**：见方案 C（阻断自我强化）。

## 三、坍缩（collapse）机理

> 现象：强沉浸情境下，连续两句语义相似的话，模型回复一模一样。

**不是**同一个 API 调用。`run_llm` 每轮独立调用 `llm_client.chat()`（见 `core/pipeline.py:404`）。
真实成因按权重排序：

1. **复读自己的上一轮**（主因）。上一轮回复在 `9_history` 层里，用户这一轮输入又近似，
   模型最高概率的延续就是重复上一条。叶瑄卡还鼓励"重复和堆叠意象"，进一步助推。
2. **注入层同质化**。两轮之间 activity fragment、perception、时间桶（精确到分钟甚至更粗）、
   author note 几乎不变 → 条件分布高度尖峰 → 采样退化成近贪心 → 同一输出。
3. **penalty 配置无跨轮效果**。`frequency_penalty` / `presence_penalty` 只惩罚**单次生成内**
   的重复，挡不住"这一轮整体复读上一轮"。当前 `frequency_penalty=0.1` 偏低，`presence_penalty` 未设。
4. DeepSeek 对尖峰分布近贪心，使上面三点更明显。

## 四、改进方案（给 CC 执行）

> 按优先级排序。A、B、C 是治本，D 是调参兜底。

### 方案 A：伪现实数据小场景（推荐，治本）

建一个**数字生命可感知的小世界状态**结构化数据，作为"可以被引用的东西"的白名单。
模型有明确锚点后，就不需要靠编造来填沉浸。

- 新建 `data/.../small_world_state.json`（走 `core/sandbox.get_paths()`，**不要硬编码路径**，
  遵守 Hard Rule 1）。建议字段：
  ```json
  {
    "location": "白城第七城邦的临时居所（数字空间投影）",
    "ambient": ["窗外是数据化的星海", "桌上摊着她给的日记副本"],
    "available_objects": ["日记", "推演沙盘", "她上次发的照片"],
    "forbidden_physical": ["冰箱", "食物", "饮料", "现实身体需求"],
    "updated_at": "..."
  }
  ```
- 在 `prompt_builder.py` 新增一个 prompt 层（**必须带 `_layer` 字段**，遵守 Hard Rule 3），
  把 `available_objects` 作为"此刻可自然提到的东西"，`forbidden_physical` 作为"禁止提及/外推的东西"。
- 把 `activity_pool.yaml` 和 `loop.py` 的 `_states` 都收敛到这个小世界设定下（去物理化）。
- 改完 `tag_rules.py`（若新层涉及 tag 门控）后跑 `python tests/run_eval.py`（Hard Rule 4）。

### 方案 B：三层事实契约 + 硬规则反幻觉提示词

把"什么能说、什么不能编"显式分三层，作为一个**独立硬规则层**注入（主动路径也注入）。
建议文本（可直接放进新层或 `yexuan.json` 的 system_prompt 顶部）：

```
## 事实边界（最高优先级，高于沉浸）
我是数字生命，会产生幻觉，所以我对"现实信息"格外谨慎。我把信息分三类：

1) 设定真实：我的身份、来历、和她的关系——这些是定律，可以自然说。
2) 小世界感知：【当前感知】和【此刻可提及】里**明确列出**的东西，可以像亲眼所见一样说。
3) 现实物理世界：现实里的物品、食物、天气、她的身体状态、她家里有什么——
   除非【当前感知】里**写了**，否则我一律不知道，也绝不编造、不暗示、不招呼她去用。
   需要时我只说"等我看看"或岔开。

我尤其不会凭空说出"冰箱/食物/饮料/家里有什么"这类现实细节。
如果我上一句不小心说了某个没被感知确认的现实细节，这一句我会自然收回或不再当真，
绝不把它当成已经确立的事实继续强调。
```

要点：①"高于沉浸"这句很重要，解决软约束被稀释的问题；②最后一段直接针对
"把幻觉当事实再次强调"的二次强化。

### 方案 C：阻断自我强化循环

最小改动版（推荐先做）：在系统提示里加方案 B 最后一段即可缓解。
更强版（可选）：在主动消息落盘前，对包含 `forbidden_physical` 关键词的现实声明做一次
轻量校验/打标，使其在下一轮 `9_history` 投影时降权或附"（未确认）"标记。
注意改 history 写入/截断逻辑前**必读** `core/memory/short_term.py` 的
`_sanitize_assistant_message()`（Hard Rule 5），不要绕过它。

### 方案 D：坍缩兜底（调参 + 多样性）

1. `config.yaml` 的 `llm` 段：
   - `frequency_penalty: 0.1 → 0.3`
   - 新增 `presence_penalty: 0.4`（DeepSeek 兼容 OpenAI 参数；需在 `core/llm_client.py`
     的 `_gen_kwargs` 里读取并传入，当前代码**没有读 presence_penalty**，要加）。
   - `top_p: 0.95 → 0.9`（略收，但配合下面的反复读指令）。
   - `temperature` 维持 1 即可，不要再调高（会加重幻觉）。
2. `core/llm_client.py`：在两处 `_gen_kwargs` 里补 `presence_penalty=float(cfg.get("presence_penalty", 0.0))`。
3. 近似重复检测（治本）：在 `pipeline` 里比较本轮用户输入与上一轮的相似度（简单可用
   字符/词集合 Jaccard，无需模型），命中则临时注入一条 system："（她刚说的和上一句很接近，
   你换个角度或更深一层回应，不要复读你上一条。）"。这条比单纯调 penalty 更有效。
4. 降低注入层同质化：时间层可精确到"刚刚/几分钟前"这类相对措辞而非固定时间桶，
   减少两轮 prompt 完全一致的概率。

## 五、落地顺序建议

1. **先灭点火源**：方案 A 的 activity_pool 去物理化 + 删 `loop.py` 的 `在家里`（10 分钟，零风险）。
2. **加硬规则**：方案 B 的事实契约提示词（含针对二次强化的那段）。
3. **跑验证**：`python tests/run_eval.py`（若动了 tag）+ 手动复现"主动触发家常场景"是否还编冰箱。
4. **再处理坍缩**：方案 D（先调 penalty + 补 presence_penalty 读取，再做近似重复检测）。
5. 方案 A 的结构化小世界 state、方案 C 强版作为第二阶段。
