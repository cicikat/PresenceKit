## 1. cc-tasks 施工单工作流固化 → **新技能**

**信号分组**(9cd18380、5336bb5a、c732fdf8、495b8d1b、c48c4c29、6f771df6、7471f2e5、ce63ad41、d316d1d4、156de4ef、d291db9c 等):
- "Cowork 诊断 → 写 cc-tasks 施工单 → 用户人肉丢给 CC/Codex → 回来口头报完成 → Cowork 只看逻辑验收(不跑测试)"这一循环几乎出现在每个会话,是核心工作流,但其约定完全靠每次现场摸索。
- 工单体例(中文、根因+文件行号+可直接贴的代码、验收标准)、并行/前置标注("没有前置的直接发,可并行的单独指出")、"一个修复一个 commit"、"行号可能漂移,先读实际代码"等纪律每次都要人工重复写进提示词。
- 工单依赖顺序靠口头约定;多条 CC lane 的冲突分析(同文件会打架)每次现场重做。
- cc-tasks 一次性工单堆积在根目录且被 git 跟踪,发布前需人工甄别。

**决策:新技能 `cc-brief`**
- 输入:问题描述/诊断结论;输出:符合既定体例的施工单(现状核对 file:line → 分仓步骤 → 验收 → 并行/前置标注 → doc-sync hook 提醒)。
- 内置规则:自动做同文件冲突分析并给出 lane 分配;完成的工单自动移入 `cc-tasks/archive/`。
- 配套改 CLAUDE.md:把"CC 跑测试、Cowork 看逻辑验收"的分工写成一句话。

## 2. 沙盒文件截断 / 弯引号 / 陈旧视图 → **修复(CLAUDE.md + hook)**

**信号分组**(0b51cf17、91b370cd、6dd29c0b、eb00b772、eabebe3b、8d5c5eee、1af88412、1d9f38d9、759cd605):
- 挂载文件在沙盒中过期/截断至少 6 个会话:pytest/tsc 读到旧快照全线误报;Edit 工具有截断前科和弯引号污染前科(B5 已两次回潮,把中文引号写进 Python 源码导致 SyntaxError)。
- Read 工具与 bash 对同一文件视图不一致;stale .pyc 让测试跑旧字节码,需手动 touch。
- 各会话已各自发明自救套路:无害 Edit 触发重新同步、脚本整段搬运代替 Edit、从 git HEAD 恢复、compileall 全仓扫描——但从未固化,每次重新踩坑。

**决策:修复**
- CLAUDE.md 增加一节「沙盒文件已知坑」:① 怀疑文件不完整时用 bash `wc -l` + `tail` 对照;② 大文件改动用脚本写整文件,不用 Edit;③ Python 改动后跑 `python -m py_compile`;④ 测试行为诡异先删 `__pycache__`。
- 新增 PostToolUse hook:对 Edit/Write 过的 `.py` 自动跑 `py_compile` + 弯引号检测(`[""'']`),失败即拦截。这直接消灭 B5 回潮类事故。

## 4. 用户偏好固化 → **修复(CLAUDE.md / 全局配置)**

**信号分组**(495b8d1b、0e9a5080、5336bb5a、608509bc、e0cfb964、156de4ef、3636d7bb、9443f288、1af88412):
- 「用中文回复」被人工重申 ≥2 次。
- 「别问我,直接推进,你替我拍板」≥3 次;「尽量减少一来一回,批量输出提示词」≥2 次。
- 「不要全仓 grep」≥2 次(对搜索成本/噪音敏感)。
- 成本敏感:「你太贵了」「不消耗这里的 tokens 了」——偏好精准小范围操作。

**决策:修复** — 在三个仓的 CLAUDE.md(或全局用户偏好)加一节「协作偏好」:中文回复;默认自主拍板、只在不可逆决策时提问;交付物一次性批量输出并标注并行/前置;定位问题用精准路径搜索,禁全仓 grep;验证优先静态核对,测试交用户/CC 本机跑。

## 5. 文档漂移定期对账 → **新自动化**

**信号分组**(8d5c5eee、1af88412、0599fe09、3636d7bb、f8258cad):
- docs/memory.md 字段名与实现不符(用户困惑"文档明明是我执行之后更新的"——文档写的是设计稿而非实现);CLAUDE.md 路径表落后 S6 布局;契约文档标着"后端是否已实现?待确认"长期未对账;某仓整套文档是从另一仓复制的错误文档;功能做一半(recall_trace 落盘但无面板入口)无人记得。

**决策:新自动化** — 建定期任务(或 skill `doc-audit`):抽查 docs/ 中的 file:line 引用、字段名、路径表与实际代码是否一致,输出漂移清单。已有 doc-sync Stop hook 管"改码必改文档",这个补的是反向核对(文档≠代码的存量债)。

## 7. 「静默失败」代码审查红线 → **修复(CLAUDE.md)**

**信号分组**(940091c3、3325e9ac、c48c4c29、0e9a5080、d291db9c):
- 627 处 `except Exception`、约 97 处 `except: pass`;索引损坏吞错返回 `{}`;日记快照 empty 不清;配置缺失静默 fallback 到错误目录;缺文件静默 fallback 到 yexuan(串号风险)。同族缺陷至少 5 次成为排查根因。

## 8. 小步 commit 纪律 → **修复(CLAUDE.md)**

**信号分组**(4c64ca81、1d9f38d9、1cc9f248):
- 用户自述"不喜欢 commit,每次攒一大坨,cc 顺手 commit 思考了两千多秒";159 个未提交改动长期悬空、21 个文件(含 1800 行变更)未入库就继续动架构;事故后无法区分"哪些坑是谁填的"。

## 10. 远控断连 / 额度中断 → **无操作(仅记录 workaround)**
