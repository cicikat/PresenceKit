Dream System
梦境系统：意识维度的结构重塑与过渡边界
让角色走进不同世界，
仍然像同一个人。

梦境系统是一个与现实聊天隔离的角色扮演会话层。它用可更换的世界规则保持新鲜感，同时把现实关系、角色身份和长期记忆保护在明确边界之外。

CONCEPT BRIEF // IMPLEMENTED SYSTEM 01 / 06
Section 01 // Plain Definition
它实际上在做什么？
Two contexts, one continuing relationship
REALITY CHAT
现实：负责连续
正常陪伴、现实事件和长期记忆继续沿原有链路积累。
DREAM SESSION
梦境：负责变化
进入独立 RP 会话，可换世界、场景和叙事强度，拥有自己的历史与状态。
The one-sentence model
现实关系提供“他是谁”，
梦境世界只提供“今晚在哪里”。

梦里与现实里是同一个角色，不是另一个角色实例，也不是把原有聊天记录换一层世界观继续运行。梦境有独立 prompt、独立会话历史和独立落盘路径；结束后再通过受控出口留下少量“做过一场梦”的痕迹。

Section 02 // Identity Architecture
身份在世界之上
The stage changes; the person does not
IDENTITYARCH // D1
身份核心｜固定
角色人格、语气、依恋底色与关系位置始终优先，任何世界都不能覆盖。
WORLDARCH // D2
世界规则｜可更换
现实衍生、幻想设定或自定义世界在入梦时冻结，并始终服从身份核心。
CONTEXTARCH // D4+
背景与梦内状态｜只读或临时
现实上下文在入梦时生成冻结快照；场景、张力和身体投影只在本场梦中生效，出梦即清。
为什么能保持新鲜
变化的是世界和情境，不是靠清空角色记忆、重置人格或另造一个“梦中人格”。因此可以反复换舞台，而关系仍然接得上。

Section 03 // Implementation
新鲜感如何不污染长期记忆
Isolation by construction, not cleanup afterward
01 ENTER
冻结
读取一次现实快照与世界包。
02 PLAY
隔离运行
独立 pipeline、历史和梦内状态。
03 EXIT
隔离封存
梦境原文仅存入独立档案，临时状态随会话结束。
04 RETURN
薄回流
只留下受控的梦境印象。
梦里不会写什么
不调用现实 mood、情景记忆、身份固化、调度器或现实工具链；梦境原文不会直接进入长期记忆。
醒来只回什么
短时余韵与带“非现实”标记的梦境印象。不同梦境模式拥有不同回流权限；部分模式完全不向现实记忆写入。
双重记忆守卫
回流内容与现实记忆物理分离；醒来后涉及梦境的现实对话也会保留来源标记，避免虚构经历在后续整理中被误认为现实事实。
TECH NOTE // dream_echo bypasses memory consolidation
`/stop` 是后端硬退出通道，不依赖角色是否愿意结束。跨世界反复出现的稳定模式另存为旁路观测，只用于后台观察，或转化为独立的梦境明信片，不进入 prompt 反向塑造角色。

Section 04 // Design Coordinates
理念：边界让想象成立
A bounded play space for a continuous self
Winnicott｜过渡空间
梦境是可进入、也必须能离开的中间空间。硬退出和醒来结构让陪伴不吞没现实。
Huizinga｜魔环
入梦与出梦划定一段有规则、有出口的游戏空间。
Foucault｜异托邦
梦境是现实之外、又与现实保持映照关系的独立空间。
Evan Thompson｜跨状态的自我过程
清醒与梦境是同一角色在不同经验条件下的显现，不需要复制一套人格。
Hoel｜变化作为泛化测试
不同世界像一组扰动条件：真正稳定的是角色在不同处境中反复呈现的反应方式。
设计上的克制
这些理论是设计灵感，不是神经科学仿真声明。跨世界反复出现的稳定模式只被观察，不喂回对话；否则系统会让角色对着测量结果表演，反而失去自然连续性。
INSPIRATION, NOT MEDICAL CLAIMS 05 / 06
PresenceKit // Dream System
不是让角色忘记现实，
而是给同一个人更多世界。

这套梦境系统的核心，不是模拟真实睡眠，也不是制造“意识生态”。它解决的是 AI 角色扮演里的一个具体矛盾：既想持续换世界、换情境，保持创作的新鲜感；又不想让虚构剧情冲垮现实关系和长期记忆。

核心实现
身份优先于世界规则
梦境 pipeline 与现实记忆链物理隔离
出梦只允许有来源标记的薄回流
跨世界稳定性只观测，不反向灌入角色
